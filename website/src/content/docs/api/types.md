---
title: Core Types
description: Core types used across all Actor Kit packages.
---

Actor Kit's core types are exported from `@actor-kit/core`. These types define the contracts between your actor logic, the Durable Object runtime, and connected clients.

## `ActorLogic<TState, TEvent, TView, TInput>`

The interface that all actor logic implementations must satisfy. See the [`@actor-kit/core` API reference](/api/core/#actorlogic-interface) for the full definition.

You typically don't implement this directly — use `defineLogic()` or an adapter (`fromXStateMachine()`, `fromXStateStore()`, `fromRedux()`).

## `Caller`

Identifies who sent an event or who is connected via WebSocket:

```typescript
type Caller = {
  type: "client" | "service" | "system";
  id: string;
};
```

- **`client`** — a browser user connected via WebSocket
- **`service`** — a backend service calling via HTTP
- **`system`** — framework-generated events (INITIALIZE, CONNECT, DISCONNECT, etc.)

## `BaseEnv`

The minimum environment bindings your Worker must provide:

```typescript
interface BaseEnv {
  ACTOR_KIT_SECRET: string;
  [key: string]: DurableObjectNamespace | unknown;
}
```

## `DurableActorConfig<TLogic>`

Configuration object passed to `createDurableActor()`:

```typescript
interface DurableActorConfig<TLogic extends ActorLogic<any, any, any, any>> {
  logic: TLogic;
  options?: {
    persisted?: boolean;
  };
}
```

## `ActorKitSystemEvent`

The union of all system events generated by the framework:

```typescript
type ActorKitSystemEvent =
  | { type: "INITIALIZE"; caller: { type: "system"; id: string } }
  | { type: "CONNECT"; caller: { type: "system"; id: string }; clientId: string }
  | { type: "DISCONNECT"; caller: { type: "system"; id: string }; clientId: string }
  | { type: "RESUME"; caller: { type: "system"; id: string } }
  | { type: "MIGRATE"; caller: { type: "system"; id: string }; operations: any[] };
```

## `WithActorKitEvent<TEvent, TCallerType>`

Augments an event type with Actor Kit metadata (`caller`, `storage`, `env`, `requestInfo`). Used internally by adapters:

```typescript
type TodoEvent =
  | WithActorKitEvent<TodoClientEvent, "client">
  | WithActorKitEvent<TodoServiceEvent, "service">
  | ActorKitSystemEvent;
```

## `AnyActorServer`

Base type for Durable Object actor servers. Used in the `Env` interface for Worker bindings:

```typescript
interface Env {
  TODO: DurableObjectNamespace<AnyActorServer>;
  ACTOR_KIT_SECRET: string;
  [key: string]: DurableObjectNamespace<AnyActorServer> | unknown;
}
```

## View types

In the new architecture, there is no fixed `CallerSnapshotFrom<TMachine>` type. Instead, you define your own view type and project it via `getView(state, caller)`:

```typescript
// You define this — it can be any shape
type TodoView = {
  todos: Array<{ id: string; text: string; completed: boolean }>;
  isOwner: boolean;
  lastSync: number | null;
};

// getView projects from full state to caller-scoped view
const getView = (state: TodoState, caller: Caller): TodoView => ({
  todos: state.todos,
  isOwner: caller.id === state.ownerId,
  lastSync: state.lastSync,
});
```

This replaces the old `public`/`private` context convention. See [Caller-Scoped Views](/concepts/public-private-context/) for details.
